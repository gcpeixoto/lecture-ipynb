Métodos numéricos usando Python (SciPy)¶

Visão geral¶

O ncleo da linguagem Python (incluindo as bibliotecas padro) fornece funcionalidades suficiente para realizar as tarefas de pesquisa computacional. No entanto, existem bibliotecas Python dedicadas (de terceiros) que oferecem funcionalidades estendidas:

  • ferramentas numricas para tarefas freqentes

  • que so convenientes

  • e mais eficientes em termos de tempo de processamento e requisitos de memria do que as funcionalidades do Python operando sozinhas.

Ns enumeramos trs desses mdulos, em particular:

  • o mdulo NumPy fornece um tipo de dado especializado para operar numericamente com vetores e matrizes (este o tipo array fornecido pelo NumPy) alm de ferramentas de lgebra linear.

  • o mdulo Matplotlib (tambm conhecido como Pylab) fornece recursos de plotagem e visualizao e

  • o mdulo SciPy (SCIentific PYthon), que fornece uma infinidade de algoritmos numricos (introduzido neste captulo).

Muitos dos algoritmos numricos disponveis atravs do SciPy e NumPy so fornecidos por bibliotecas compiladas estabelecidas que so muitas vezes escritas em Fortran ou C. Eles, por sua vez, sero executados muito mais rpido do que o cdigo Python puro (interpretado). Como regra geral, esperamos que o cdigo compilado seja duas ordens de magnitude mais rpido do que o cdigo Python puro.

Voc pode usar a funo de ajuda para cada mtodo numrico para descobrir mais sobre a origem da implementao.

Integração numérica¶

O SciPy fornece uma srie de rotinas de integrao. Uma ferramenta com a finalidade de resolver integrais $I$ do tipo $$I = \int_a^b f(x) \, dx$$ fornecida pela funo quad() do mdulo scipy.integrate.

Ela usa como argumento de entrada, a funo a ser integrada $f(x)$ ("integrando") e os limites inferior $a$ e superior $b$. Ela retorna dois valores (em uma tupla): o primeiro o resultado calculado e o segundo uma estimativa do erro numrico desse resultado.

Aqui est um exemplo que produz esta sada:

In [1]:
from math import cos, exp, pi
from scipy.integrate import quad

# funo a ser integrada
def f(x):
    return exp(cos(-2 * x * pi)) + 3.2

# chamamos quad para integrar f de -2 a 2
res, err = quad(f, -2, 2)

print("O resultado numrico  {:f} (+-{:g})".format(res, err))

O resultado numrico  17.864264 (+-1.55113e-11)

Note que quad() usa os parmetros opcionais epsabs e epsrel para aumentar ou diminuir a preciso de sua computao. (Use help(quad) para saber mais.) Os valores padro so epsabs = 1.5e-8 e epsrel = 1.5e-8. Para o prximo exerccio, os valores padro so suficientes.

Resolvendo equações diferenciais ordinárias (EDOs)¶

Para resolver uma EDO do tipo $$\frac{dy}{dt}(t) = f(y,t)$$

com uma dada condio inicial $y(t_0) = y_0$, podemos usar a funo odeint do SciPy. Aqui est um programa de exemplo auto-explicativo (useodeint.py) para encontar

$$y(t) \quad \mathrm{para} \quad t \in [0,2]$$ dada a EDO : $$\frac{dy}{dt}(t) = -2yt \quad \mathrm{com} \quad y(0) = 1.$$

In [2]:
%matplotlib inline
from scipy.integrate import odeint
import numpy as N

def f(y, t):
    """este  o lado direito da EDO a ser integrada, i.e. dy/dt = f(y,t)"""
    return -2 * y * t

y0 = 1             # valor inicial
a = 0              # limites de integrao para t
b = 2

t = N.arange(a, b, 0.01)  # valores de t para
                          # os quais queremos
                          # a soluo y(t)
y = odeint(f, y0, t)  # clculo de y(t)

import pylab          # plotagem dos resultados
pylab.plot(t, y)
pylab.xlabel('t'); pylab.ylabel('y(t)')

Localização de raízes¶

Se voc tentar encontrar um $x$ tal que $$f (x) = 0,$$ ento este um problema chamado de localizao de razes. Observe que problemas como $g(x) = h(x)$ caem nesta categoria, pois voc pode reescrev-los como $f(x) = g(x) - h(x) = 0$.

Vrias ferramentas para localizao de razes esto disponveis no mdulo optimize do SciPy.

Localização de raiz pelo método de bisecção¶

Primeiro, apresentamos o algoritmo bisect, que (i) robusto e (ii) lento, mas conceitualmente muito simples.

Suponhamos que precisemos calcular as razes de $f(x) = x^3 - 2x^2$. Esta funo tem uma raiz dupla em $x = 0$ (isto trivial de ver) e outra raiz localizada entre $x = 1.5$ (pois $f(1.5) = - 1.125$) e $x = 3$ (pois $f(3) = 9$). direto ver que esta outra raiz est localizada em $x = 2$. Aqui est um programa que determina esta raiz numericamente:

In [3]:
from scipy.optimize import bisect

def f(x):
    """retorna f(x) = x^3 - 2x^2. Tem razes em
    x = 0 (raiz dupla) e x = 2"""
    return x ** 3 - 2 * x ** 2

# o programa principal comea aqui
x = bisect(f, 1.5, 3, xtol=1e-6)

print("A raiz x  aproximadamente x=%14.12g,\n"
      "o erro  menor do que 1e-6." % (x))
print("O erro exato  %g." % (2 - x))

A raiz x  aproximadamente x= 2.00000023842,
o erro  menor do que 1e-6.
O erro exato  -2.38419e-07.

O mtodo bisect() requer trs argumentos obrigatrios: (i) a funo $f(x)$, (ii) um limite inferior $a$ (para o qual escolhemos 1.5 no nosso exemplo) e (ii) um limite superior $b$ (para o qual escolhemos 3). O parmetro opcional xtol determina o erro mximo do mtodo.

Um dos requisitos do mtodo da biseco que o intervalo $[a,b]$ deve ser escolhido de tal forma que a funo seja positiva em $a$ e negativa em $b$, ou que a funo seja negativa em $a$ e positiva em $b$. Em outras palavras, $a$ e $b$ devem incluir uma raiz.

Localização de raízes usando a função fsolve¶

Um algoritmo (frequentemente) melhor (no sentido de "mais eficiente") do que o algoritmo da biseco implementado na funo generalizada fsolve() para a localizao de razes de funes (multidimensionais). Este algoritmo precisa apenas de um ponto de partida prximo provvel localizao da raiz (mas no possui convergncia garantida).

Aqui est um exemplo:

In [4]:
from scipy.optimize import fsolve

def f(x):
    return x ** 3 - 2 * x ** 2

x = fsolve(f, 3)           # one root is at x=2.0

print("A raiz x  aproximadamente x=%21.19g" % x)
print("O erro exato  %g." % (2 - x))

O erro exato  -6.66134e-15.

O valor de retorno [6] de fsolve uma matriz numpy de comprimento $n$ para um problema de localizao de razes com $n$ variveis. No exemplo acima, temos $n = 1$.

Interpolação¶

Dado um conjunto de $N$ pontos $(x_i, y_i)$ com $i = 1, 2, \dots, N$, s vezes precisamos de uma funo $\hat{f}(x)$ que retorne $y_i = f(x_i)$ e que, alm disso, fornea alguma interpolao dos dados $(x_i, y_i)$ para todo $x$.

A funo y0 = scipy.interpolate.interp1d (x, y, kind = 'nearest') realiza esta interpolao baseada em splines de ordem varivel. Observe que a funo interp1d retorna uma funo y0 que ir ento interpolar os dados x-y para qualquer dado $x$ quando chamado como y0(x).

O cdigo abaixo mostra isso, bem como os diferentes tipos de interpolao.

In [32]:
import numpy as np
import scipy.interpolate
import pylab

def cria_dados(n):
    """Dado um inteiro n, retorne n pontos
    x e valores y como um numpy.array."""
    xmax = 5.
    x = np.linspace(0, xmax, n)
    y = - x**2
    
    # faa os dados dos pontos x um tanto irregulares
    y += 1.5 * np.random.normal(size=len(x))
    return x, y

# programa principal
n = 10
x, y = cria_dados(n)

# usa uma malha mais fina e irregular para a plotagem
xfine = np.linspace(0.1, 4.9, n * 100)

# interpola com funo constante por partes (p=0)
y0 = scipy.interpolate.interp1d(x, y, kind='nearest')

# interpola com funo linear por partes (p=1)
y1 = scipy.interpolate.interp1d(x, y, kind='linear')

# interpola com funo quadrtica por partes (p=2)
y2 = scipy.interpolate.interp1d(x, y, kind='quadratic')

pylab.plot(x, y, 'o', label='dados')
pylab.plot(xfine, y0(xfine), label='constante')
pylab.plot(xfine, y1(xfine), label='linear')
pylab.plot(xfine, y2(xfine), label='quadrtica')
pylab.legend()
pylab.xlabel('x')

Ajuste de curva¶

J vimos que podemos ajustar funes polinomiais a um conjunto de dados usando a funo numpy.polyfit. Aqui, apresentamos um algoritmo de ajuste de curva mais genrico.

O SciPy fornece uma funo um pouco genrica (baseada no algoritmo de Levenberg-Marquardt) atravs do scipy.optimize.curve_fit para ajustar uma determinada funo (Python) a um conjunto de dados. O pressuposto que temos um conjunto de dados com pontos $x_1, x_2, ... x_N$ e com valores de funo correspondentes $y_i$ dependentes de $x_i$, tal que $y_i = f(x_i,\textbf{p})$. Queremos determinar o vetor de parmetros $\textbf{p} = (p_1, p_2, \dots, p_k)$ para que $r$, a soma dos resduos, seja a menor possvel:

$$r = \sum \limits_{i = 1}^N \left (y_i - f(x_i,\textbf{p}) \right)^2$$

O ajuste da curva de uso particular se os dados forem contaminados por rudos: para um dado $x_i$ e $y_i = f(x_i, \textbf{p})$, temos um termo de erro (desconhecido) $\epsilon_i$ para que $y_i = f(x_i,\textbf{p}) + \epsilon_i$.

Utilizamos o seguinte exemplo para esclarecer isso: $$f(x,\textbf{p}) = a \exp(-bx) + c, \quad \mathrm{i.e.} \quad \textbf{p} = (a, b, c)$$

In [5]:
import numpy as np
from scipy.optimize import curve_fit


def f(x, a, b, c):
    """Ajusta funo  y=f(x,p) com parmetros p=(a,b,c). """
    return a * np.exp(- b * x) + c

# cria dados aleatrios
x = np.linspace(0, 4, 50)
y = f(x, a=2.5, b=1.3, c=0.5)

# adiciona rudo
yi = y + 0.2 * np.random.normal(size=len(x))

# chama a funo de ajuste
popt, pcov = curve_fit(f, x, yi)
a, b, c = popt
print("Os parmetros timos so a=%g, b=%g, and c=%g" % (a, b, c))

# plotagem
import pylab
yfitted = f(x, *popt)   # equivalente a f(x, popt[0], popt[1], popt[2])
pylab.plot(x, yi, 'o', label='dados $y_i$')
pylab.plot(x, yfitted, '-', label='ajuste $f(x_i)$')
pylab.xlabel('x')
pylab.legend()

Observe que no cdigo-fonte acima, definimos a funo de ajuste $y = f(x)$ atravs do cdigo Python. Podemos, portanto, ajustar funes (quase) arbitrrias usando o mtodo curve_fit.

A funo curve_fit retorna uma tupla popt, pcov. A primeira entrada popt contm uma tupla dos parmetros timos (no sentido de que estes minimizam o valor $r$). A segunda entrada contm a matriz de covarincia para todos os parmetros. As diagonais fornecem a varincia dos parmetro estimados.

Para que o processo de ajuste de curva funcione, o algoritmo de Levenberg-Marquardt precisa iniciar o processo de ajuste com palpites iniciais para os parmetros finais. Se estes no forem especificados (como no exemplo acima), o valor "1.0" usado para o palpite inicial.

Se o algoritmo falhar em obter uma funo para os dados (mesmo que a funo descreva os dados razoavelmente), precisamos dar ao algoritmo melhores estimativas para os parmetros iniciais. Para o exemplo mostrado acima, podemos dar as estimativas para a funo curve_fit mudando a linha

Popt, pcov = curve_fit (f, x, yi)

para

Popt, pcov = curve_fit (f, x, yi, p0 = (2,1,0.6))

se nossos palpites iniciais fossem $a =2$,$b=1$ e $c=0.6$. Uma vez que tomamos o algoritmo "aproximadamente na rea correta" no espao de parmetros, o ajuste geralmente funciona bem.

Otimização¶

Muitas vezes precisamos encontrar o mximo ou mnimo de uma funo particular $f(x)$, onde $f$ uma funo escalar, mas $x$ poderia ser um vetor. As aplicaes tpicas so a minimizao de algumas variveis, tais como custo, risco e erro, ou a maximizao de produtividade, eficincia e lucro. Rotinas de otimizao normalmente fornecem um mtodo para minimizar uma determinada funo: se precisamos maximizar $f(x)$, criamos uma nova funo $g(x)$ que inverte o sinal de $f$, ou seja, $g(x) = - f(x)$ e minimizamos $g(x)$.

Abaixo, damos um exemplo mostrando (i) a definio da funo de teste e (ii) a chamada da funo scipy.optimize.fmin, que toma como argumento a funo $f$ a ser minimizada e um valor inicial $x_0$ a partir do qual se inicia a busca pelo mnimo, e retorna o valor de $x$ para o qual $f(x)$ (localmente) minimizado. Normalmente, a busca pelo mnimo uma busca local, i.e, o algoritmo segue o gradiente local. Ns repetimos a busca pelo mnimo para dois valores ($x_0 = 1.0$ e $x_0 = 2.0$, respectivamente) para demonstrar que, dependendo do valor de partida, podemos encontrar diferentes mnimos para a funo $f$.

A maioria dos comandos (aps as duas chamadas de fmin) no arquivo fmin1.py cria o grfico da funo, os pontos de partida para as buscas e o mnimo obtido:

In [6]:
from scipy import arange, cos, exp

import pylab

def f(x):
    return cos(x) - 3 * exp( -(x - 0.2) ** 2)

Valor de retorno de fmin¶

Note que o valor de retorno da funo fmin um array do numpy que - para o exemplo acima - contm apenas um nmero j que temos um nico parmetro (aqui $x$) para variar. Em geral, fmin pode ser usada para encontrar o mnimo em um espao de parmetros de dimenso superior se houver vrios parmetros. Nesse caso, o array numpy conteria aqueles parmetros que minimizam a funo objetivo. A funo objetivo $f(x)$ tem que retornar um escalar mesmo que haja mais parmetros, ou seja, mesmo se $x$ for um vetor.
