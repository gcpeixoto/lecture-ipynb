Matplotlib e Pylab¶

Podemos ver o Matplotlib como uma biblioteca de plotagem orientada a objetos. Pylab uma interface para o mesmo conjunto de funes que imita a interface de plotagem do Matlab.

Pylab um pouco mais conveniente de usar para plotagens fceis e o Matplotlib d um controle muito mais detalhado sobre a forma como os plots so criados. Se voc usa o Matplotlib rotineiramente para produzir figuras, aconselhamos voc a aprender sobre a interface orientada a objetos do Matplotlib (em vez da interface Pylab).

Este captulo concentra-se na interface Pylab.

Como importar tudo isso: matplotlib, pylab, pyplot, numpy¶

O sub-mdulo matplotlib.pyplot fornece uma interface orientada a objetos para a biblioteca de plotagem. Muitos dos exemplos na documentao do Matplotlib seguem a conveno de importao do matplotlib.pyplot como plt e o numpy como np. Evidentemente, a deciso sobre importar a biblioteca numpy com o mome np (como frequentemente feito nos exemplos da documentao), ou com o nome N como feito neste texto (e nos primrdios do Numeric, predecessor do NumPy), ou ainda com qualquer outro nome, uma deciso plena do usurio. Da mesma forma, uma questo de gosto pessoal se o submdulo de plotagem matplotlib.pyplot deve ser importado como plt, como feito na documentao do Matplotlib, ou como plot, se a argumentao for por uma escrita mais clara, etc.

Como sempre, um equilbrio deve ser encontrado entre preferncias pessoais e consistncia com a prtica comum ao escolher esses nomes. A consistncia com o uso comum , certamente, mais importante se o cdigo estiver propenso a ser usado por outros ou publicado.

Uma plotagem quase sempre precisa de arrays de dados numricos e por esta razo que o mdulo NumPy bastante usado: ele fornece manipulao rpida e eficiente de memria em Python.

Poderamos, portanto, ter escrito o exemplo 1a acima como o exemplo 1b (que idntico em funcionalidade ao anterior na criao da mesma plotagem):

In [3]:

y = N.sin(x)

pylab.plot(x, y)
Out[3]:
[<matplotlib.lines.Line2D at 0x105d63828>]

Pelo fato de numpy.arange e numpy.sin serem objetos j importados para o espao de nomes (conveniente) pylab, tambm poderamos ter escrito o exemplo 1c:

Isto pode ser extremamente conveniente, mas vem com um grande e "salutar" aviso:

  • Enquanto usar from pylab import * aceitvel no prompt de comando para criar grficos interativamente e analisar dados, isto nunca deve ser usado como regra de uso nos scripts de plotagem.

  • O Pylab fornece mais de 800 objetos diferentes que so todos importados para o espao de nomes global quando executamos o comando from pylab import *. Isto no uma boa prtica, e pode gerar conflitos com outros objetos que j existam ou com os que sero criados mais tarde.

  • Como uma regra de uso: nunca invoque from somewhere import * em programas que salvamos. Esta uma opo razovel no prompt de comando.

Nos exemplos a seguir, usamos a interface do pylab para as rotinas de plotagem, mas isto puramente uma questo de gosto e hbito. De nenhuma maneira, a nica opo a seguir (note que os autores do Matplotlib recomendam o estilo de importao do exemplo 1a. Salvando figuras como arquivos¶

Depois de ter criado uma figura (usando o comando plot) e adicionado legendas, rtulos, etc, voc tem duas opes para salvar a plotagem.

  1. Exibir a figura (usando show) e salv-la de forma interativa clicando no boto de salvamento .

  2. Sem exibir a figura, salv-la do seu cdigo Python diretamente para arquivo. O comando a usar savefig. O formato determinado pela extenso do nome do arquivo que voc fornece. Aqui est um exemplo (pylabsavefig.py).

In [6]:
# pylabsavefig.py

import pylab
import numpy as N

x = N.arange(-3.14, 3.14, 0.01)
y = N.sin(x)

pylab.plot(x, y, label='sin(x)')
pylab.savefig('static/data/15-meuplot.png')  # salva arquivo png 
pylab.savefig('static/data/15-meuplot.eps')  # salva arquivo eps
pylab.savefig('static/data/15-meuplot.pdf')  # salva arquivo pdf
pylab.close()

Uma nota sobre formatos de arquivo: escolha o formato de arquivo png se voc pretende incluir o seu grfico em um documento do Word ou em uma pgina web. Escolha os formatos pdf ou eps se voc pretende incluir a figura em um documento LaTeX - dependendo se voc for compilar usando latex (requer eps) ou pdflatex (pode usar pdf [melhor] ou png). Se a verso do Microsoft Word (ou outro software de processamento de texto que voc usa) puder lidar com arquivos pdf, melhor usar pdf do que png.

Tanto pdf quanto eps so formatos vetoriais de arquivo, o que significa que podemos ampliar a imagem sem que ela perca a qualidade (as linhas ainda sero ntidas). Formatos de arquivos como png, jpg, gif, tif e bmp salvam a imagem em formato de mapa de bits (bitmap), ou seja, uma matriz de valores de cor, que aparecer desfocada ou pixelizada ao ser ampliada ou impressa em alta resoluo.

Modo interativo¶

O Matplotlib pode ser executado de dois modos:

  • no interativo (padro)

  • interativo.

No modo no interativo, nenhum plot ser exibido at que o comando show() tenha sido chamado. Neste modo, o comando show() deve ser a ltima declarao do seu programa.

No modo interativo, os plots sero imediatamente mostrados depois que o comando de plotagem tiver sido chamado.

Pode-se ativar o modo interativo com o comando pylab.ion() e desativ-lo com o comando pylab.ioff(). O comando mgico %matplotlib do IPython tambm habilita o modo interativo.

Segue uma lista de outros comandos teis:

  • figure(figsize=(5,5)) configura o tamanho da figura para 5x5 (polegadas)

  • plot(x,y1,label=sin(x)) a palavra-chave label define o nome desta linha. O rtulo da linha ser mostrado na legenda se o comando legend() for usado posteriormente.

  • Note que chamando o comando plot repetidamente, voc poder sobrepor uma srie de curvas.

  • axis([-2,2,-1,1]) restringe a rea de plotagem nos limites de xmin=-2 a xmax=2 na direo x e de ymin=-1 a ymax=1 na direo y.

  • legend() este comando mostrar uma legenda com os rtulos definidos no comando plot. Experimente help(pylab.legend) para aprender mais sobre o posicionamento das legendas.

  • grid() este comando mostrar as linhas de grade em segundo plano.

  • xlabel(...) e ylabel(...) adicionam legendas aos eixos.

Alm disso, voc pode escolher diferentes estilos e espessuras de linha, cores e smbolos para os dados a serem plotados. A sintaxe muito semelhante do Matlab. Por exemplo:

  • plot(x,y,og) plotar crculos (o) na cor verde (g)

  • plot(x,y,-r) plotar uma linha (-) na cor vermelha (r)

  • plot(x,y,-b,linewidth=2) plotar uma linha azul (b) com dois pixels de espessura linewidth=2, que duas vezes mais espessa do que o padro.

import numpy as np
import matplotlib.mlab as mlab    #  comandos compativeis com Matlab
import matplotlib.pyplot as plt
import matplotlib.cm as cm        # submdulo de mapas de cor

# cria matriz Z que contm alguns dados interessantes
delta = 0.025
x = y = np.arange(-3.0, 3.0, delta)
X, Y = np.meshgrid(x, y)
Z = mlab.bivariate_normal(X, Y, 3.0, 1.0, 0.0, 0.0)

# ajuste de espaamento dos subplots para evitar sobreposio
aux = 1.0
subplots_adjust(wspace=aux, hspace=aux)

Nx, Ny = 2, 3
plt.subplot(Nx, Ny, 1)  # prximo plot ser mostrado no
                        # primeiro subplot como uma 
                        # matriz Nx x Ny de subplots

plt.imshow(Z, cmap=cm.jet)  # mapa de cor padro 'jet'
plt.title("mapa de cor: jet")

plt.subplot(Nx, Ny, 2)   # prximo plot para segundosubplot
plt.imshow(Z, cmap=cm.jet_r)  # mapa de cor jet reverso
plt.title("mapa de cor: jet_r")

plt.subplot(Nx, Ny, 3)
plt.imshow(Z, cmap=cm.gray)
plt.title("mapa de cor: gray")

plt.subplot(Nx, Ny, 4)
plt.imshow(Z, cmap=cm.hsv)
plt.title("mapa de cor: hsv")

plt.subplot(Nx, Ny, 5)
plt.imshow(Z, cmap=cm.gist_earth)
plt.title("mapa de cor: gist_earth")

plt.subplot(Nx, Ny, 6)

# produz isolinhas reduzindo o nmero de cores para 10
mycmap = cm.get_cmap('jet', 10)    # 10 cores discretas
plt.imshow(Z, cmap=mycmap)
plt.title("mapa de cor: jet\n(10 cores apenas)")

Visualizando dados em dimensões superiores¶

Muitas vezes, precisamos entender os dados definidos em posies no espao 3D. Os dados em si so muitas vezes um campo escalar (como uma temperatura) ou um vetor 3D (como velocidade ou campo magntico), ou ocasionalmente um tensor. Por exemplo, para um campo vetorial 3D (definido como $\vec{f}(\vec{x})$, onde $\vec{x} \in \! \mathbb{R}^3$ e $\vec{f}(\vec{x}) \in \! \mathbb{R}^3$), poderamos desenhar uma seta 3D em cada ponto (grade) no espao. comum que esses conjuntos de dados dependam do tempo.

A biblioteca provavelmente mais comumente utilizada nas cincias e engenharia para visualizar tais conjuntos de dados provavelmente o VTK, Visualization ToolKit (http://vtk.org). Esta uma importante biblioteca C++ com interfaces para linguagens de alto nvel, incluindo o Python.

Pode-se chamar essas rotinas diretamente do cdigo Python ou escrever os dados no disco em um formato que a biblioteca VTK pode ler (chamados arquivos vtk) e, em seguida, usar um programa como Mayavi, ParaView ou VisIt para ler esses arquivos de dados e manipul-los (com uma interface grfica). Todos estes trs esto usando a biblioteca VTK internamente e podem ler arquivos vtk.

Este pacote muito adequado para visualizar campos estticos e temporais 2D e 3D (campos escalares, vetoriais e tensoriais).

Escrevendo arquivos vtk a partir do Python (pyvtk)¶

Uma pequena, mas poderosa biblioteca Python est disponvel em https://code.google.com/p/pyvtk/. Ela permite criar arquivos vtk a partir de estruturas de dados do Python muito facilmente.

Dada uma malha de elementos finitos ou um conjunto de dados de diferenas finitas em Python, pode-se usar o pyvtk para escrever esses dados em arquivos e, em seguida, usar uma das aplicaes de visualizao listadas acima para carregar os arquivos vtk para exibi-los e investig-los.
